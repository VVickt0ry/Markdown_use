
Markdown 是一款轻量级纯文本标记语言，通过简单的符号标记就能实现文本的格式排版
写出来的内容简洁易读，还能轻松转换为 HTML、PDF等格式,语法跨平台通用



1. 标题的缩进
用#表示层级，#数量越多，标题级别越低，#和标题文字之间必须加空格



2. 文字的样式
*斜体文本* 或 _斜体文本_ 
**粗体文本** 或 __粗体文本__
***粗斜体文本*** 
~~删除线文本~~ 



3. 引用
> 一级引用：核心备注
>> 二级引用：补充说明
>>> 三级引用：细节补充
>
> 回到一级引用：引用内空行只需加一个>，再按回车即可

# 单行引用适合标注备注、参考说明

# 多行引用
# 写法1：每行都加>
# 写法2：仅第一行加>，之后的内容缩进写

如果引用后想直接跟代码块、水平线、表格等独立元素，不用空行，这些元素会自动切断引用格式
如果引用后不空行、也不插独立元素，直接写普通内容，会被当成引用的一部分
大于号和引用内容之间必须加空格




4. 水平线分割线
用***/---/___都可以，单独占一行




5. 列表
分为无序列表和有序列表，在符号和文字之间加空格，嵌套时缩进即可，适合写步骤、要点
# 无序列表（用-/*/+都可以）
- 第一点
- 第二点
  - 第二点的子点
- 第三点

# 有序列表（数字+ .  数字无需按顺序，Git会自动排序）
1. 克隆仓库
2. 修改代码
   1. 暂存代码
   2. 提交代码
3. 推送远程




6. 代码块
代码块是Markdown里专门用来放代码的，指定语言自动语法高亮
把代码和普通文字分开，GitHub、VSCode里会自动高亮颜色，复制的时候不会带乱码


6.1 单行代码块（在esc下面用反引号`包裹，反引号和内容之间无空格）

6.2 多行代码块
开头和结尾格三个反引号```，开头的反引号后面可指定语言，如python/shell/plaintext
语言标识和反引号之间无空格，代码在反引号之间，保留原始换行和缩进
# 指定 python 作为语言标识，编辑器会自动对关键字、字符串、注释高亮
# 指定 shell 作为语言标识，对命令关键字、参数高亮
# 指定 plaintext 作为语言标识，无语法高亮，仅原样展示纯文本


6.3 提示
在VSCode中，Markdown Preview Enhanced 插件支持直接运行代码块，把光标放在代码块内
按Shift+Enter运行当前代码块，运行后会在代码块下方直接显示输出结果/图表


6.4 Markdown不支持代码块内嵌套代码块，如需展示反引号，可在反引号前加 \ 转义
比如 \`git clone\`，实际上是单行代码块

1. 链接
用于添加网页链接、仓库链接、文档内跳转


1.1 行内式链接
格式: [链接文字](链接地址 "鼠标悬停提示")    


1.2 参考式链接
参考式链接是把外部链接地址抽离出来，在文档末尾统一管理的写法,多处引用同一链接时使用
格式: 正文[标记][别名] + 文末[别名]: 地址 "提示"


1.3 锚点链接
锚点链接在同一文档内跳转，跳转到指定标题
格式: [跳转文字](#目标标题)
规则:目标标题转小写；目标标题中的空格替换为短横线 - ；去掉目标标题的特殊符号
多级标题直接用最终的标题文本转换即可



2. 脚注
脚注是超链接，不会打乱正文排版，点击会跳转到文末对应的脚注，点击返回标记，又能回到正文位置
正文加标记[^数字] + 文末写注释[^数字]: 补充内容
 



3. 图片
语法和链接类似，开头多一个！，支持本地图片和网络图片
格式: ![图片描述](图片地址 "鼠标悬停提示")     描述为空则写![]()

网络图片  ![GitHub图标](https://github.githubassets.com/images/logo.png "Logo")
仓库图片，图片要与README.md同目录  ![Markdown笔记](markdown_note.png "学习笔记")
调整图片大小：GitHub专用<img src="地址" width="数值" height="数值">，Markdown兼容HTML




4. 表格
适合整理对比性内容，竖线是分隔列，短横线分隔表头和内容，冒号指定对齐，默认左对齐
| Git命令         | 功能说明                     |
| :-------------- | :--------------------------- |
| git branch      | 查看本地分支                 |

说明：:在-左侧=左对齐，两侧=居中，右侧=右对齐
注意：短横线数量至少1个，列数要和表头一致
